# Skill — beta-rollout (GTM)

## When to use
새 기능 / 제품의 closed beta → open beta → GA rollout 계획.

## Inputs
- Launch checklist (앞 단계)
- 베타 후보 사용자 / segment
- Risk class (HIGH 일수록 long beta)

## Steps
1. **stage 정의**: closed (10-50) → open (100-1000) → GA. stage 당 기간 / 사용자 수.
2. **entry criteria**: 다음 stage 진입 — 어떤 metric / 신호?
3. **exit / rollback**: 각 stage 의 fail signal + rollback path.
4. **feedback loop**: 사용자 piracy / 인터뷰 / NPS — 어떻게 모음?
5. **참가자 selection**: 페르소나 covers / engagement 수준 / 다양성.
6. **incentive**: 베타 사용자 — 인센티브 / 인지도 / early access.
7. **legal / privacy**: 베타 약관 + opt-out + 데이터 활용 동의.

## Output
- Rollout plan (1-2 페이지):
  - 3 stage — entry / exit / 기간 / 사용자 수
  - Feedback collection 방법
  - 참가자 selection rubric
  - Legal sign-off

## Quality bar
- Entry / exit criteria 정량
- Feedback loop cadence (weekly / per-stage)
- Rollback path 명시
- 참가자 — 대표 페르소나 cover
