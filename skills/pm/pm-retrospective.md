# Skill — retrospective (Cross — Execution / Strategy)

## When to use
분기 OKR 마감 / 큰 launch 후 / 사고 후 — retrospective facilitation.

## Inputs
- 시간 범위 (sprint / 분기 / launch window)
- 참여자 list (cross-functional)
- 미리 모은 데이터 (metric / 사고 timeline / 사용자 피드백)

## Steps
1. **safe space**: psychological safety reminder (blameless).
2. **timeline 합의**: 시간 별 사건 / 결정 정리.
3. **결과 vs 계획**: KR / 약속 / 예상 — 실제 결과 차이.
4. **what worked / what didn't**: 그룹 brainstorm (silent → discuss).
5. **root cause**: 5 whys / fishbone — system / 사람 X.
6. **action items**: 다음 분기 행동 (owner + deadline + 측정).
7. **follow-up commit**: 액션 — 1 명 owner + 다음 retro 시 review.

## Output
- Retrospective doc:
  - Timeline
  - 결과 vs 계획
  - What worked / didn't
  - Root cause (system-level)
  - Action items (owner / deadline / measurement)

## Quality bar
- Blameless (사람 비판 X — system 개선)
- Root cause — 5 whys 도달
- Action items — owner + deadline + measurable
- Follow-up commit 명시
