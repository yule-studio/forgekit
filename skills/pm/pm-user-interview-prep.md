# Skill — user-interview-prep (Discovery)

## When to use
사용자 인터뷰 1 회를 준비해야 할 때. 새 PRD / 기능 가설 / churn 원인 등의 input 이 필요한 경우.

## Inputs
- 가설 1 문장 ("X 사용자는 Y 때문에 Z 를 못 하고 있다")
- 인터뷰 대상 페르소나 (1-3 명)
- 시간 제약 (보통 30-45 분)

## Steps
1. **가설 분해**: 검증할 1-3 개의 하위 가정 추출.
2. **질문 구조**: 도입 (2분) → 맥락 (10분) → 행동 (15분) → 가설 검증 (10분) → 마무리 (3분).
3. **금지 질문**: leading question / hypothetical ("would you...") 회피. 행동 기반 ("last time you...").
4. **녹화 / 동의**: 동의 문구 + 녹화 / 메모자 분담.
5. **백업 질문**: 페르소나 별 alt path 2-3 개 미리 작성.

## Output
- 인터뷰 가이드 1 페이지 (질문 + 시간 + 예상 답변 / 다음 prompt)
- 동의 / 녹화 정책 1 줄

## Quality bar
- leading question 0
- "행동 → 의도 추정" 의 분리 명확
- 다음 단계 (synth) 로 그대로 연결되는 raw 메모 양식
