# Skill — persona-mapping (Discovery)

## When to use
타겟 사용자를 1-3 개 페르소나로 정리해야 할 때. PRD 의 "for whom" 명확화 또는 마케팅 메시지 정렬용.

## Inputs
- 인터뷰 / 분석 데이터
- 비즈니스 segmentation 가정
- (옵션) 기존 페르소나 v0

## Steps
1. **JTBD 추출**: 페르소나 별 "Job To Be Done" 1-2 개 식별.
2. **demographics ≠ behavior**: 나이 / 직급보다 — 행동 / 동기 / pain 중심.
3. **변별력 검증**: 두 페르소나가 의사결정 다른가? 같으면 합친다.
4. **anti-persona**: "타겟이 아닌 사용자" 도 1 명 작성 — scope drift 방지.
5. **검증 가능성**: 각 페르소나 — 어떤 데이터 / 인터뷰가 contradicting?

## Output
- 페르소나 카드 (1 페이지 / 페르소나):
  - 이름 + 한 줄 요약
  - JTBD 1-2
  - Pain / 현재 workaround
  - 의사결정 trigger
  - anti-persona 비교

## Quality bar
- behavioral / JTBD 기반 (demographic only X)
- 1-3 명 (너무 많으면 변별력 없음)
- 의사결정 implication 명시 ("이 페르소나는 X 를 우선시한다 → PRD 의 Y 결정")
