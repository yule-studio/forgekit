# Skill — roadmap-quarterly (Strategy)

## When to use
분기 로드맵 수립 / 갱신. PRD → KR → 시간 축 mapping.

## Inputs
- 분기 OKR
- backlog (PRD draft / idea pool)
- 자원 (engineering capacity / dependency)

## Steps
1. **outcome → output mapping**: KR 별 — 가능한 output 후보 brainstorm.
2. **prioritisation**: RICE / ICE / Cost of Delay 적용 (pm-prioritisation-rice 참고).
3. **공약 vs 탐색**: 분기의 60-70% — committed, 30-40% — exploratory.
4. **시간 축**: month / sprint 단위. 의존 / 순서 명시.
5. **non-goals**: 이번 분기 하지 않을 일 명시 (scope drift 방지).
6. **stakeholder 검토**: engineering capacity / marketing campaign / sales 약정과 충돌 없는지.

## Output
- 로드맵 1 페이지:
  - Theme / OKR alignment
  - committed items (PRD ref + month)
  - exploratory items (가설 + 단위 검증)
  - non-goals 5+ 개

## Quality bar
- KR 와 명시적 mapping
- engineering capacity 검증 (over-commit 금지)
- non-goal 명시
- exploratory 의 검증 기준 명시
