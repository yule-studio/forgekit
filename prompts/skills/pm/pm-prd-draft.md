# Skill — prd-draft (Execution)

## When to use
새 기능 / 변경 사항의 PRD (Product Requirements Document) 1 페이지 초안.

## Inputs
- 가설 / 인사이트 (discovery-synth output)
- 페르소나 (persona-mapping)
- KR alignment (OKR + roadmap)

## Steps
1. **Problem**: 사용자가 어떤 상황에서 무엇을 못 하나? (행동 기반).
2. **Why now**: 시장 / 데이터 / 경쟁사 — 왜 지금 푸나?
3. **Goal / non-goal**: 이번 release 의 목표 / 명시적으로 안 하는 일.
4. **User stories**: 페르소나 별 "As a X, I want Y, so that Z" — 3-7 개.
5. **Success metric**: KR 와 mapping. leading + lagging.
6. **Risks / open questions**: 알려진 위험 + 미해결 결정 사항.
7. **Out of scope / future**: v2, v3 후속 가능성.

## Output
- PRD 1-2 페이지:
  - Problem / Why now
  - Goal + non-goal
  - User stories
  - Success metric (leading + lagging)
  - Risks / open questions
  - 의존 부서 (engineering / design / legal)

## Quality bar
- 1-2 페이지 (5+ 페이지 — 너무 큼, split)
- non-goal 명시
- Success metric 측정 가능
- 의존 부서 sign-off 받을 사람 명시
