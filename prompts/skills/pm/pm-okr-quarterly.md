# Skill — okr-quarterly (Strategy)

## When to use
분기 OKR (Objectives + Key Results) 수립 / 갱신.

## Inputs
- 회사 / 부서의 annual strategy
- 지난 분기 OKR retrospective
- 외부 제약 (예산 / 자원 / 의존 부서)

## Steps
1. **Objective**: 1-3 개. 정성적 / 영감을 주는 / 분기 후 "달성 yes/no" 명확.
2. **Key Result**: O 당 2-4 개. 정량적 / 측정 가능 / 60-70% confidence (stretch).
3. **leading vs lagging**: KR 중 최소 1 개는 leading (early signal).
4. **부서 OKR 정렬**: 회사 OKR → 부서 OKR — explicit mapping.
5. **dependency 명시**: KR 마다 — 의존 부서 / 사람 / 시스템 list.
6. **체크인 cadence**: 주간 / 격주 / 월간 — 명시.

## Output
- OKR sheet (1 페이지):
  - Objective + 한 줄 narrative
  - KR list (현재값 / 목표값 / 측정 방법 / owner)
  - dependency / risk
  - 체크인 schedule

## Quality bar
- KR 60-70% confidence (100% — too easy, 30% — moonshot)
- 측정 방법 / 데이터 source 명시
- 의존 / 위험 explicit
- Lagging 만 — 안 됨 (leading 최소 1 개)
