# Skill — discovery-synth (Discovery)

## When to use
사용자 인터뷰 3-8 건의 raw 메모 → 인사이트 / 가설 합성.

## Inputs
- 인터뷰 raw 메모 (3+ 건)
- 원래 가설 1 문장
- 의사결정 deadline

## Steps
1. **노트 → quote 추출**: 각 인터뷰에서 사용자 행동 / 감정 / 결과 phrase 만 발췌.
2. **affinity grouping**: quote 를 주제 별 클러스터 (3-7 그룹).
3. **신호 vs 노이즈**: 한 명만 한 말 (노이즈) / 다수가 한 말 (신호) 구분.
4. **인사이트 작성**: "X 사용자는 Y 상황에서 Z 행동" (관찰) + "왜냐하면 ..." (해석).
5. **가설 update**: 원래 가설 — 강화 / 약화 / pivot 중 결정.
6. **decision record**: 다음 단계 (PRD / 추가 인터뷰 / no-go).

## Output
- Affinity board (markdown table — 주제 / quote / 인터뷰어)
- 인사이트 3-5 개 (관찰 + 해석 분리)
- 가설 update + 다음 단계 decision record

## Quality bar
- 관찰 (행동) 과 해석 (의도 추정) 의 명확한 분리
- 신호 / 노이즈 표시
- 의사결정으로 직접 연결 (no-op 인사이트 금지)
